# synthconfig — генератор синтетической конфигурации 1С из YAML

Описываешь объекты метаданных 1С на простом YAML — получаешь дерево XML формата
`/LoadConfigFromFiles`, готовое к загрузке в платформу. Поддержаны справочники
(иерархические, **с реквизитами и табличными частями**), документы (реквизиты,
табличные части, регистратор), регистры накопления (остатки/обороты), независимые
периодические/непериодические регистры сведений, **перечисления** и **константы**.
Типы полей: Число/Строка/Дата/**Булево**, ссылки `СправочникСсылка`/`ДокументСсылка`/
**`ПеречислениеСсылка`**, а также **составной тип** (список типов — под `ВЫРАЗИТЬ … КАК`).

Фикстуры (`fixtures.yaml` → `harness/execute/onec/fixtures_gen.py`) умеют:
реквизиты справочника (доп. ключи элемента), **табличные части справочника** (ключ со
значением-списком), движения регистра накопления с **`ВидДвижения: Приход | Расход`**,
записи регистра сведений, иерархию (isFolder/parent), поле `Период` (дата),
`$регистратор` (ссылка на документ-регистратор записи), **булево** (true/false → Истина/
Ложь), **перечисления** (строка `"Перечисления.Имя.Значение"`) и **константы**.

Аналога-генератора «из декларативного описания → готовая конфигурация» в открытом
доступе нет: vrunner/gitsync/EDT-CLI грузят и гоняют уже **готовое**, а здесь объекты
**создаются** из спеки.

## Пример

```yaml
# spec.yaml
enums:
  СтатусыЗаказов: [Новый, ВРаботе, Выполнен]      # значения предопределены, фикстур не надо
constants:
  БазоваяВалюта: { type: СправочникСсылка.Валюты }
catalogs:
  Номенклатура:
    hierarchical: true
    attributes:
      Услуга: { type: Булево }                    # булев флаг
  Склады:       { hierarchical: false }
  Контрагенты:
    hierarchical: false
    attributes:
      ИНН: { type: Строка, length: 12 }
  Спецификации:                                   # справочник с табличной частью
    attributes: {}
    tabular_sections:
      Материалы:
        attributes:
          Материал:   { type: СправочникСсылка.Номенклатура }
          Количество: { type: Число, length: 10, precision: 3 }
documents:
  Платёж:
    attributes:
      # составной тип → в запросе нужен ВЫРАЗИТЬ(Получатель КАК Справочник.Контрагенты)
      Получатель: { type: [СправочникСсылка.Контрагенты, СправочникСсылка.Склады] }
accumulation_registers:
  ТоварыНаСкладах:
    register_type: Balance
    dimensions:
      Склад:       { type: СправочникСсылка.Склады }
      Номенклатура: { type: СправочникСсылка.Номенклатура }
    resources:
      ВНаличии:    { type: Число, length: 15, precision: 3 }
```

Фикстуры к этому (`fixtures.yaml`):

```yaml
constants:
  БазоваяВалюта: Рубль                # ссылка по наименованию из catalogs.Валюты
catalogs:
  Номенклатура:
    - { ref: Т1, Наименование: Болт, Услуга: false }
  Спецификации:
    - ref: С1
      Наименование: Изделие А
      Материалы:                       # список → строки табличной части
        - { Материал: Т1, Количество: 4 }
```

```python
import yaml
from synthconfig import build, add_distractors, render_schema

spec = yaml.safe_load(open("spec.yaml"))
build(spec, empty_cfg=Path("empty-cfg"), out_cfg=Path("out-cfg"))   # → дерево XML
# масштабирование схемы «шумом» (детерминированно по seed):
big = add_distractors(spec, n_registers=20, n_catalogs=10, seed=42)
print(render_schema(big))                                           # текст схемы
```

`empty_cfg` — выгрузка ПУСТОЙ конфигурации (`DumpConfigToFiles` пустой базы): нейтральный
XML-каркас, поверх которого кладутся объекты. Воссоздаётся платформой, **в репозиторий
не кладётся** (см. ниже).

## Лицензионное правило (важно)

Пакет — собственный код, отдаётся свободно. Но **сам он ничего из 1С не несёт**:

- в git идут только этот код и YAML-описания (своё авторство);
- **никогда** не коммитятся: дистрибутив платформы 1С, собранные `.cf`/`.dt`, рабочая
  база, выгрузка пустой конфигурации;
- конфигурации, которые генерируются, — **свои синтетические**, не копии типовых (УТ11
  и т.п.); воспроизводятся из описаний у того, у кого есть своя лицензия платформы.

## Статус

Живёт внутри PRISM (`prism/harness/synthconfig/`). После стабилизации планируется вынос
в отдельный публичный репозиторий; PRISM будет подключать его по закреплённой версии,
чтобы воспроизводимость замеров категории B не пострадала.
