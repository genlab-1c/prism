# Changelog

Заметные изменения PRISM. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/);
версии по [SemVer](https://semver.org/lang/ru/) на пару «набор задач + определение метрики».
**v1.0.0** — первый публичный релиз исполняемого бенчмарка.

## [1.0.0] — 2026-06-17

### Прогон и модели
- **Первый авто-прогон Уровня 1:** 5 моделей (Alice, Gemini 2.5 Flash Lite, DeepSeek
  V4-Flash, Qwen3-235B, GLM-4.7 Flash) на 25 задачах A и 15 B — сравнительный лидерборд в README.
- **Function calling для Yandex Foundation Models** (нативные toolCallList/toolResultList):
  агентный сбор метаданных категории B заработал для Yandex-моделей; OpenRouter уже поддерживал.
- Банк задач вырос до **40** (A1–A25, B1–B15).
- **Провал генерации (напр. HTTP 404) → «не измерено»**, а не балл 0.

### Метрика и оценка
- **Плавная оценка на лидерборде.** Оси Смысл (M) и Платформа (P) показываются точно
  (доля × 10), без мёртвых зон 6-балльной шкалы. Ступенька в 6 баллов остаётся только для
  сверки с экспертом (см. [Как это работает](docs/how-it-works.md) → «Шкала оценок»).
- **Q на лидерборде** как грубый общий балл — вторичен; главный результат — вектор
  `[S, M, O, P]`.
- **Профиль по навыкам** (срезы по тегам) вынесен на публичный лидерборд: с чем модель
  сильна/слаба, модель-vs-модель, всегда с числом задач в срезе.

### Производительность
- **Параллельный скоринг кандидатов** (env `PRISM_CONCURRENCY`, по умолчанию ≤4) — на
  баллы не влияет (проверено бит-в-бит: 1 поток и 4 дают идентичный результат).
- Опциональные лимиты контейнера 1С `PRISM_ONEC_MEMORY` / `PRISM_ONEC_CPUS` (по умолчанию
  не заданы → поведение как было).

### Тесты
- Покрыты ось P (`score_p`), парсер результата категории B (граница осей P↔M), агрегация
  срезов по тегам.

### Документация
- Документация переписана на простой язык; добавлена страница «Как это работает»;
  «Что умеет сейчас» и «Честные границы» отвязаны от мартовского эксперимента.
- Лицензия **MIT** (`LICENSE`), `CITATION.cff`, `.env.example`.
- Добавлены этот CHANGELOG и [«Ключевые решения»](docs/decisions.md).

### Данные
- Мартовские результаты удалены — будут перегенерированы на актуальных моделях.
- Удалён легаси `tasks/tasks_category_B.yaml` (промпты живут в `task.yaml` каждой задачи).
