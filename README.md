# PRISM — бенчмарк кодогенерации 1С кода с помощью ИИ (Концепт, превью)

Отправная точка бенчмарка **PRISM**: методика **SMOP** (Syntax · Meaning · Optimization ·
Platform) и эксперимент марта 2026 в том виде, в каком они были созданы и опубликованы.
Это **концепт**, а не релиз продукта: экспертный опросник. Исходная точка для построения зрелого исполняемого бенчмарка первой версии (v1.0.0) — PRISM.

**Метка:** `paper-baseline` (снапшот, вне продуктовой SemVer-линии).
**Публичный приоритет методики** — репозиторий научной работы (март 2026):
https://github.com/4dand/1c-ai-codegen-research-paper
(статья «Экспериментальная оценка эффективности ИИ в генерации кода для доменно-специфичных
платформ (на примере 1С:Предприятие 8)», Андреев Д.С.).

## Состав

```
metric/smop_criteria.yaml        # критерии SMOP (шкала 0/2/4/6/8/10, банды S/M/O/P)
tasks/                           # задачи марта: категории A (алгоритмические) и B (УТ11)
  tasks_category_A.yaml          #   + конфиги прогона (models.yaml, experiment.yaml)
  tasks_category_B.yaml
results/                         # данные эксперимента: 30 генераций, 3 модели
  experiment_A_*.json            #   сырые результаты генерации
  experiment_B_*.json
  evaluations/                   #   экспертная разметка SMOP (expert_01)
  code_outputs/                  #   сгенерированный код (.bsl)
```

## Что было

- 10 задач (5 алгоритмических + 5 платформенных под УТ11), 3 модели
  (Claude Opus 4.5, GPT-5.2 Codex, Gemini 3 Flash), генерация через OpenRouter.
- Экспертная разметка по SMOP.
- Известные ограничения зафиксированы в статье (раздел 3.4) и устранены в переработке.

## Установка

Быстро через `make` (парадная дверь — `make` без аргументов покажет все цели):

```bash
make setup        # .venv + инструменты осей на хост (= venv + tools)
make check        # целостность контрактов/заданий/эталонов/инструментов
make score        # авто-оценка L1 свежих генераций
make images       # собрать docker-образы инструментов (для режима docker)
```

Ниже — то же вручную. Окружение бенчмарка — два слоя: Python-харнесс и внешние инструменты осей.

**1. Python-харнесс** (загрузчики, скореры, расчёт Q):

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"        # PyYAML + pydantic, dev — pytest
pytest                         # самопроверка контрактов и логики
```

**2. Инструменты осей** (исполнение/анализ кода кандидатов). Бинари не коммитятся — ставятся локально:

```bash
./tools/get-onescript.sh        # ось M: oscript в tools/onescript/ (исполнение тестов)
./tools/get-bsl-ls.sh           # ось S: BSL Language Server (статический парсер), нужен java 21+
# ось M в песочнице:
docker build -t prism-onescript -f docker/onescript.Dockerfile .
```

Режим исполнения M — `PRISM_RUNNER` (`local` по умолчанию | `docker`); путь к Java для S —
`PRISM_JAVA` при нестандартном. Любой инструмент недоступен → его ось «не измерена»
(`score=None`), а не нулём — `compute_q` исключает её из среднего.

**Точка входа** `prism` (после `pip install -e`):

```bash
prism check                       # целостность: контракты, задания, эталоны, инструменты
prism score                       # авто-оценка L1 свежих генераций по изданию core → results/auto/
prism score --edition core --experiment results/experiment_A_20260301_125458.json
```

Без установки — `python3 -m harness.cli <команда>`.

## Что дальше (зрелая версия)

Переработка переводит SMOP с экспертного опросника на **исполняемую двухуровневую оценку**:
авто-оракулы (компилятор, тесты, схема метаданных) + экспертный оверлей; категория B
исполняется против синтетических конфигураций. Первый публичный релиз исполняемого
бенчмарка получит номер **v1.0.0**.
