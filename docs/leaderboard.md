# Лидерборд — какая нейросеть лучше пишет код на 1С?

PRISM отвечает на этот вопрос числом, а не мнением. Ниже — **авто-прогон Уровня 1**:
средний вектор по осям SMOP на одних и тех же задачах, напрямую — модель против модели.
Главный результат — **вектор** `[S, M, O, P]`, а не одно число; **Q** вторичен (грубое
ранжирование).

Таблицы регенерируются из оценок командой `prism docs` (источник правды —
`results/auto/*_auto_l1.json`), руками их не правят.

## Категория A — алгоритмика

Чистый BSL: сортировки, разбор строк, работа с датами и коллекциями. Исполняется в
OneScript. Ось **P** (платформа) здесь неприменима. Оптимальность (**O**) проверяем **на
деле**: подаём алгоритму большой объём входных данных и смотрим, как растёт число операций.
Так быстрый алгоритм сразу отличается от медленного — например, перебор всех пар вместо
поиска по словарю.

## Категория B — платформа

Платформенные задачи: остатки на складе, списание по FIFO, цены, себестоимость, валюты.
Исполняются в настоящей 1С без интерфейса против синтетической базы. Оптимальность (**O**)
здесь оцениваем по самому коду — ищем типовые тяжёлые ошибки (например, запрос к базе внутри
цикла). Ось **P** (платформа) показывает, обращается ли модель к реальным объектам базы или
выдумывает несуществующие.

## Как читать и как пополнить

- Цифры сравнимы **модель против модели на одних и тех же задачах**: важна не абсолютная
  величина, а разница и профиль.
- Свежий лидерборд из готовой оценки — мгновенно: `uv run prism leaderboard` (без пересчёта).
- Прогнали свою модель? Поделитесь прогоном: `uv run prism submit` упакует результат с
  хешем совместимости версии бенчмарка — см. [Как запустить](cli.md) и
  [Как участвовать](https://github.com/genlab-1c/prism/blob/main/CONTRIBUTING.md).
