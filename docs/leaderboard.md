средний вектор по осям SMOP на одних и тех же задачах, напрямую — модель против модели.
Главный результат — **вектор** `[S, M, O, P]`, а не одно число; **Q** вторичен (грубое
ранжирование).

`results/auto/*_auto_l1.json`), руками их не правят.

## Категория B — платформа

Платформенные задачи: остатки на складе, списание по FIFO, цены, себестоимость, валюты.
Исполняются в настоящей 1С без интерфейса против синтетической базы. Оптимальность (**O**)
здесь меряем исполнением: наращиваем базу и считаем, как растёт число обращений к СУБД. Кто
берёт данные одним запросом — обращений почти не прибавляет; кто лезет в базу в цикле —
прибавляет на каждую строку. Задачам, для которых нагрузочный профиль ещё не написан, балл по
O ставит разбор кода — по типовым антипаттернам вроде запроса в цикле. Ось **P** (платформа)
показывает, обращается ли модель к настоящим объектам базы или выдумывает несуществующие.

## Где ломается код у каждой модели

Оценка показывает, насколько код хорош. Эта таблица — **на каком шаге он ломается**.
Каждая полоса — все попытки модели, разложенные по итогу (в сумме 100%):
🟩 решено · 🟨 неверный ответ (код отработал, но результат не тот) · 🟧 ошибка выполнения
(упал при запуске или работе) · 🟥 не компилируется. Столбец «самая частая поломка» —
что у этой модели чаще всего ломается среди нерешённых: что чинить в первую очередь.

## Профиль по навыкам — с чем модели справляются лучше, а с чем хуже

Каждая задача помечена тегами (навыки, конструкции 1С) — отсюда срез «модель × тег».
Видно **что именно** даётся модели, а не только общий балл. Теги, на которые пришлось меньше
трёх задач, в таблицу не выносятся — по паре задач вывод делать рано.

Теплокарта: цвет по уровню балла — 🟩 хорошо (≥7) · 🟨 средне (4–7) · 🟥 слабо (<4).

## Как читать и как пополнить

- Цифры сравнимы **модель против модели на одних и тех же задачах**: важна не абсолютная
  величина, а разница и профиль.
- Свежий лидерборд из готовой оценки — мгновенно: `uv run prism leaderboard` (без пересчёта).
- Прогнали свою модель? Поделитесь прогоном: `uv run prism submit` упакует результат с
  хешем совместимости версии бенчмарка — см. [Как запустить](cli.md) и
  [Как участвовать](https://github.com/genlab-1c/prism/blob/main/CONTRIBUTING.md).
