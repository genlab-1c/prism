# PRISM

**Исполняемый бенчмарк кодогенерации 1С** по методике **SMOP** — четыре оси качества:
**S**yntax · **M**eaning · **O**ptimization · **P**latform.

Переработка переводит SMOP с экспертного опросника на **двухуровневую оценку**:
авто-оракулы Уровня 1 (компилятор, тесты, схема метаданных) + экспертный оверлей
Уровня 2. Числа и правила живут в YAML-контрактах, не в коде.

!!! note "Статус"
    Концепт-превью. Первый публичный релиз исполняемого бенчмарка получит номер **v1.0.0**.

## Принцип

Код читает данные, данные не живут в коде. Пороги, банды, веса осей — в
`metrics/*.yaml`; оси, применимые к категории, — в конституции; задачи — в
`tasks/<категория>/<id>/`. Если инструмент оси недоступен (OneScript, BSL LS),
ось помечается «не измерена» (`score=None`), а не нулём — и исключается из среднего Q.

## Установка

Два слоя окружения: Python-харнесс и внешние инструменты осей.

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"          # харнесс: PyYAML + pydantic, dev — pytest

./tools/get-onescript.sh         # ось M: исполнение скрытых тестов (OneScript)
./tools/get-bsl-ls.sh            # оси S/O: статический парсер (BSL LS, нужен java 21+)
```

## Быстрый старт

```bash
prism check                      # целостность: контракты, задания, эталоны, инструменты
prism score                      # авто-оценка L1 свежих генераций по изданию core
```

Подробнее о командах — [CLI (prism)](cli.md). Об устройстве и потоке данных —
[Архитектура](architecture.md).

## Оси и инструменты

| Ось | Что измеряет | Инструмент (категория A) | Категория B |
|-----|--------------|--------------------------|-------------|
| **S** | компилируемость | BSL Language Server | компилятор 1С |
| **M** | смысл (исполнение тестов) | OneScript | 1С + YAXUNIT |
| **O** | оптимальность (perf-антипаттерны) | BSL Language Server | BSL LS |
| **P** | платформа (метаданные) | — N/A | схема синтетической конфигурации |

**Q** = среднее по применимым **измеренным** осям (`mean_of_applicable`).
