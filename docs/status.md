# Что умеет сейчас

Здесь честно: что в PRISM уже работает, чем это подтверждено и что в планах. Где функция
сырая — так и написано.

!!! note "В двух словах"
    Автоматическая оценка (Уровень 1) работает для обеих категорий задач: код
    компилируется, прогоняется тестами и запускается в реальной 1С. Слой эксперта
    (Уровень 2) описан, но пока ждёт разметки — для него нужны живые специалисты.

Правила и пороги живут в файлах метрики (`metrics/`), а где наша оценка надёжна, а где
ещё нет — на странице [Честные границы](validity.md). Здесь — только карта состояния.

## Что уже работает

| Способность | Состояние | Чем подтверждено |
|---|---|---|
| Категория A: оси Синтаксис, Смысл, Оптимальность автоматически | ✅ рабочее | `prism check`: все эталоны категории A набирают M=10; тесты зелёные |
| Категория B: запуск кода в реальной 1С | ✅ рабочее | `prism check`: все эталоны категории B проходят «(1С) S=10 · M=10 · P чисто» |
| Синтаксис категории B проверяет компилятор 1С | ✅ рабочее | ошибки ловит настоящий компилятор (`/CheckModules`), а не приблизительный анализатор |
| Платформа (P) из запуска: «пережил ли код контакт с базой» | ✅ рабочее | обращение к несуществующему полю ловится и отделяется от просто неверного ответа |
| Кто виноват — код или окружение | ✅ рабочее | код не компилируется → 0; сбой окружения (нет Docker и т.п.) → «не измерено», а не 0 |
| Сборка учебной базы из описания задачи | ✅ рабочее | база собирается из описания и загружается в 1С без ошибок |
| Полный прогон оценки (S · M · O · P + общий балл) | ✅ рабочее | одной командой `prism score` для категорий A и B |
| Уровень 2: эксперты и согласие с машиной | ⏳ ждёт разметки | платформа для экспертов готова; согласие пока не посчитать — нужен не один эксперт |

## Из чего состоит

- **Задачи** — двух категорий: алгоритмические (`tasks/category_a/`) и платформенные
  (`tasks/category_b/`). Каждая платформенная задача несёт описание своей учебной базы,
  тестовые данные, проверки и эталонное решение. Эталоны проверяются настоящим запуском
  в `prism check`.
- **Метрика** — правила оценки в `metrics/`: ось «Платформа» считается запуском кода,
  устаревшая проверка «на глаз» убрана.
- **Запуск кода** — OneScript для категории A; 1С без интерфейса (в Docker) для категории
  B; плюс автоматическая сборка учебной базы.
- **Оценка** — модули `harness/score/` (по одному на каждую ось); один запуск в 1С даёт
  сразу и Смысл (M), и Платформу (P).

## Результаты

### Авто-прогон Уровня 1 (7 моделей)

Средний общий балл **Q** на одних и тех же задачах (полные векторы S/M/O/P — в README).
Таблица регенерируется из оценок командой `prism docs`:

<!-- prism:status:lb -->
| Модель | Q · категория A | Q · категория B |
|---|:---:|:---:|
| **DeepSeek V4-Flash** | **9.02** | **6.23** |
| Alice AI LLM | 8.05 | 5.00 |
| Gemini 2.5 Flash Lite | 7.90 | 4.50 |
| GLM-4.7 Flash | 7.86 | 5.77 |
| Qwen3-235B-A22B | 7.58 | 4.96 |
| YandexGPT 5 Lite | 7.11 | 4.93 |
| YandexGPT 5.1 Pro | 6.88 | 4.62 |
<!-- /prism:status:lb -->

*(A — 25 алгоритмических задач, B — 15 платформенных; ранги по A и B немного различаются —
напр. GLM на B второй.)*

**Что показал прогон:** **DeepSeek V4-Flash** — лучший на обеих осях при копеечной цене;
**Qwen3-235B** (235 млрд параметров) — в хвосте обеих, то есть размер не предсказывает
доменный навык 1С. Ось **Платформа** различает модели по тому, выдумывают ли они метаданные.

!!! note "Это первый авто-прогон"
    Одна генерация на задачу, дешёвые модели, L1 без экспертной сверки. Границы
    применимости — [Честные границы](validity.md). Полные срезы по навыкам даёт `prism score`.

## Что в планах (по приоритету)

1. **Первый открытый прогон** на актуальных моделях → первые честные цифры и лидерборд.
2. **Больше задач.** Банк пока небольшой; рост числа и разнообразия задач — главный
   приоритет (как добавить — в [CONTRIBUTING](https://github.com/genlab-1c/prism/blob/main/CONTRIBUTING.md)).
3. **Скорость категории B.** На практике ~10–20 секунд на кандидата (пустая конфигурация
   кэшируется), не минуты. Прогоны идут параллельно (env `PRISM_CONCURRENCY`) — общее время
   масштабируется по ядрам, на баллы это не влияет. Общий кэш пустой конфигурации есть. Кэш собранной базы
   под каждую задачу **отложен**: параллелизм уже снял главное, а сам кэш хрупок (модуль
   «Тесты» завязан на имя функции кандидата → пришлось бы догружать модули частично, риск
   для бит-в-бит точности). Вернёмся к нему, если реальный прогон окажется слишком медленным.
4. **Проверки категории B на YAXUNIT** — это стандартный для 1С формат модульных тестов
   (сейчас формат свой; смысл проверок при переходе сохранится).
5. **Уровень 2 с участием двух и более экспертов** → первое настоящее измерение согласия
   машины и эксперта.
